## ⏳ 남은 개발 (선택)

아래는 **기획서에는 있지만 아직 구현 안 된** 항목입니다. 필요할 때만 진행하면 됩니다.

| 항목 | 설명 | 우선순위 |
|------|------|----------|
| **이벤트 수정 전용 URL** | 현재는 상세 페이지(`/admin/events/[id]`)에서 수정. 별도 `/admin/events/[id]/edit` 페이지는 없음. (기능적으로는 충분) | 낮음 |
| **보상 선택 (CHOICE)** | 이벤트 보상이 "포인트 vs 쿠폰 중 선택"인 경우, 승인 후 사용자가 [보상 선택] 버튼으로 POINT/COUPON 택 1 → 지급. 현재는 관리자 승인 시 단일 보상만 처리. | 중간 |
| **ALWAYS 빈도 제한** | 상시 이벤트 "1일 1회 / 1주 1회 / 1월 1회" 참여 제한. DB `frequency_limit` 컬럼·마이그레이션은 있음. 화면·참여 가능 여부 로직 미연동. | 중간 |
| **MAU 지표** | 관리자 대시보드 "MAU". 월별 접속 수 집계용 테이블/로그 없으면 "준비 중" 또는 제외. | 낮음 |
| **알림** | "칭찬이 도착했습니다" 등 알림 발송. 설계만 되어 있음. | 낮음 |
| **V.Honors 랭킹** | PRD Phase 3. 개인/팀 랭킹 페이지. | Phase 3 |
| **Vertex AI 이미지 분류** | 챌린지 인증샷 자동 식별 등. Phase 3. | Phase 3 |
