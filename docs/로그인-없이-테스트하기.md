# 로그인 없이 테스트하는 방법 (비개발자용)

로그인 기능을 아직 쓰지 않아도, **5만 포인트**를 가진 상태로 기부·대시보드를 테스트할 수 있습니다.  
아래 순서대로만 하면 됩니다.

---

## 준비물

- **Supabase** 프로젝트가 이미 만들어져 있어야 합니다.
- **프로젝트 폴더**가 컴퓨터에 있고, 터미널에서 `npm run dev`로 서버를 켤 수 있어야 합니다.

---

## 1단계: Supabase에서 “테스트용 사용자” 만들기

데이터베이스에 **“게스트 테스트”** 라는 가상 사용자를 한 명 넣는 단계입니다.  
이 사용자에게 5만 포인트가 들어갑니다.

> **궁금한 점:** **+ 로 새로 만드는 게 아니라**, 지금 쓰는 **「V.Together Platform」 그대로** 사용하면 됩니다.  
> SQL Editor에서 + 또는 「New query」는 **새 프로젝트가 아니라**, “SQL을 입력할 **빈 칸 하나 더 열기**”입니다.  
> 그 빈 칸에 SQL을 붙여 넣고 Run 하면 **지금 있는 V.Together Platform 데이터베이스에** 테스트 유저가 **추가**됩니다.

### 1-1. Supabase 사이트 접속

1. 브라우저에서 [https://supabase.com](https://supabase.com) 접속 후 로그인합니다.
2. **지금 쓰는 프로젝트**(예: V.Together Platform)를 클릭해 들어갑니다.  
   - **+ 로 새 프로젝트를 만들 필요는 없습니다.**

### 1-2. SQL Editor 열기

1. 왼쪽 메뉴에서 **「SQL Editor」** 를 클릭합니다.
2. **「New query」** 버튼(또는 + 아이콘)을 눌러 **새 SQL 입력 창**을 엽니다.  
   - 이건 “새 프로젝트”가 아니라, **SQL을 쓸 빈 칸 하나 더 여는 것**입니다.  
   - 실행하면 **지금 있는 V.Together Platform Schema(데이터베이스)에** 테스트 유저가 추가됩니다.

### 1-3. SQL 문 넣고 실행하기

1. 프로젝트 폴더를 연 다음, 아래 파일을 **메모장이나 VS Code**로 엽니다.  
   - 경로: `docs` → `migrations` → **`003-guest-test-user.sql`**
2. **파일 안에 있는 글자 전체**를 **복사**합니다.  
   (처음 `--` 로 시작하는 줄부터 마지막 `;` 까지 전부)
3. Supabase **SQL Editor**의 빈 칸에 **붙여넣기** 합니다.
4. 오른쪽 아래 **「Run」** 버튼(또는 재생 아이콘)을 클릭합니다.
5. 아래에 **「Success. No rows returned」** 같은 문구가 나오면 **성공**입니다.  
   - 에러가 나오면: `docs/database-schema.sql` 을 아직 한 번도 실행하지 않았다면, **먼저 그 파일 전체를 복사해서 SQL Editor에서 실행**한 뒤, 다시 003 파일을 실행해 보세요.

이렇게 하면 DB에 **“guest-test”** 라는 테스트 사용자(5만 P 보유)가 생깁니다.

---

## 2단계: Service Role Key 복사하기

Supabase가 “비로그인 상태에서도 이 테스트 사용자로 기부할 수 있게” 허용하려면, **비밀 키**를 프로젝트에 알려줘야 합니다.

1. Supabase 왼쪽 메뉴에서 **「Project Settings」**(톱니바퀴 아이콘)를 클릭합니다.
2. **「API」** 메뉴를 클릭합니다.
3. 화면 아래쪽 **「Project API keys」** 부분을 찾습니다.
4. **「service_role」** 이라고 적힌 줄에서 **「Reveal」** 또는 **「Copy」** 를 눌러 **키 값 전체**를 복사합니다.  
   - **주의:** 이 키는 절대 다른 사람에게 보내거나, SNS·채팅에 붙여넣지 마세요.  
   - **anon (public)** 키가 아니라 **service_role** 키를 복사해야 합니다.

---

## 3단계: 프로젝트에 환경 변수 넣기

프로젝트가 “테스트 모드”로 동작하려면, **환경 변수** 두 개를 넣어야 합니다.

### .env.local 이 뭔가요?

- **프로젝트 폴더 안에 있는 설정 파일**입니다.  
  “이 프로젝트는 이런 값들을 쓸게요”라고 적어 두는 **메모 파일**이라고 보면 됩니다.
- **이름이 `.env.local`** 이고, **맨 앞에 점(.)** 이 있어서  
  - Mac: 보통 **숨김 파일**로 보일 수 있고  
  - Windows: “숨긴 항목”을 켜야 보일 수 있습니다.  
  **vtogether** (또는 프로젝트 이름) 폴더를 연 곳에서, **같은 단계**에 있어야 합니다.  
  - 예: `vtogether` 폴더 안에 `app` 폴더, `docs` 폴더, `package.json` 이 보이는 곳에 `.env.local` 이 있어야 합니다.
- **비밀 키·주소 같은 것**을 적어 두는 파일이라, **Git에 올리지 말고** 로컬에서만 쓰는 게 좋습니다. (이미 `.gitignore` 에 들어가 있을 수 있습니다.)

**없으면:** 그냥 **같은 폴더에 새 파일**을 만들고, 이름을 **정확히** `.env.local` 로 지어서 저장하면 됩니다.  
**이미 `.env` 만 있다면:** 그 파일을 쓰거나, 새로 `.env.local` 을 만들어서 그 안에만 아래 내용을 넣어도 됩니다.

### 3-1. `.env.local` 파일 찾기 / 만들기

1. **vtogether** (프로젝트 최상위) 폴더를 엽니다.  
   - VS Code로 프로젝트를 연 경우: 왼쪽에서 **맨 위 폴더**가 vtogether 인지 확인합니다.
2. **`.env.local`** 파일이 있는지 봅니다.  
   - **숨김 파일 보기:** Mac은 `Cmd + Shift + .` , Windows는 탐색기에서 “숨긴 항목” 체크.  
   - **없으면:** 새 파일 만들기 → 이름을 **`.env.local`** 로 저장 (앞의 점까지 정확히).

### 3-2. 내용 넣기

1. `.env.local` 파일을 **메모장이나 VS Code**로 엽니다.
2. **맨 아래**에 아래 두 줄을 **그대로** 추가합니다.  
   (이미 같은 이름이 있으면 그 줄만 수정합니다.)

```env
GUEST_TEST_USER_ID=guest-test
SUPABASE_SERVICE_ROLE_KEY=여기에_방금_복사한_service_role_키_붙여넣기
```

3. **「여기에_방금_복사한_service_role_키_붙여넣기」** 부분을 **지우고**, 2단계에서 복사한 **service_role 키 전체**를 붙여넣습니다.  
   - 따옴표는 넣지 않아도 됩니다.  
   - 줄 끝에 빈 칸이 없도록 합니다.
4. 파일을 **저장**합니다.

예시 (실제 키는 더 깁니다):

```env
GUEST_TEST_USER_ID=guest-test
SUPABASE_SERVICE_ROLE_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.xxxx...
```

## 4단계: 서버 다시 켜기

환경 변수는 **서버를 켤 때**만 읽기 때문에, 한 번 **끄고 다시 켜야** 적용됩니다.

1. 터미널에서 서버가 돌아가고 있다면 **Ctrl + C** 로 종료합니다.
2. 아래 명령어로 다시 실행합니다.

```bash
npm run dev
```

3. 브라우저에서 **http://localhost:3000** 을 엽니다.

---

## 5단계: 화면에서 확인하기

1. **로그인 버튼은 누르지 않고** 그냥 메인 페이지만 봅니다.
2. 오른쪽 위에 **「게스트 (테스트)」** 또는 **「Eco Keeper」** 같은 표시가 보여야 합니다.
3. **사용 가능 포인트**에 **50,000 P** (또는 5만 P)가 보이면 성공입니다.
4. **기부하기** 버튼을 눌러서 금액을 선택하고 기부까지 진행해 보면, 포인트가 줄어드는 것도 확인할 수 있습니다.

---

## 자주 나오는 문제

### “SQL 넣었는데 안 돼요” / 포인트가 0으로 보여요

**순서대로 확인해 보세요.**

1. **SQL 실행 시 에러가 났는지**
   - Run 누른 뒤 **빨간색 에러 메시지**가 안 나왔는지 확인하세요.
   - **“relation \"users\" does not exist”** → `docs/database-schema.sql` 을 **먼저** 전체 복사해서 Run 한 뒤, 다시 003 파일을 실행하세요.
   - **“permission denied”** / **“row-level security”** → Supabase SQL Editor에서 실행한 것이 맞는지 확인하고, 같은 프로젝트(V.Together Platform)에서 실행했는지 보세요.

2. **테이블에 유저가 들어갔는지**
   - Supabase 왼쪽 메뉴 **Table Editor** → **users** 테이블 클릭.
   - **user_id** 가 **guest-test** 인 행이 있는지, **current_points** 가 **50000** 인지 확인하세요.
   - 없으면 003 SQL을 **처음부터 끝까지** 다시 복사해서 붙여 넣고 Run 해 보세요. (마지막 `SELECT` 줄까지 넣으면 실행 후 결과로 1건이 보입니다.)

3. **.env.local 설정**
   - 프로젝트 폴더의 **.env.local** 에 아래가 **정확히** 있는지 확인하세요.
     - `GUEST_TEST_USER_ID=guest-test` (앞에 공백 없이, 따옴표 없이)
   - 저장했는지 확인하세요.

4. **서버 재시작**
   - .env 를 수정했다면 **반드시** 터미널에서 서버를 **Ctrl+C 로 끄고**, `npm run dev` 로 다시 켜야 적용됩니다.

### “사용자 정보를 찾을 수 없습니다” / 포인트가 0으로 보여요

- 위 **“SQL 넣었는데 안 돼요”** 의 2번, 3번, 4번을 순서대로 확인하세요.
- Supabase **Table Editor** → **users** 에 **guest-test** 행이 있고, .env 에 **GUEST_TEST_USER_ID=guest-test** 가 있고, 서버를 다시 켰다면 동작해야 합니다.

### “테스트 모드: .env에 SUPABASE_SERVICE_ROLE_KEY를 설정해주세요” 라고 나와요

- **3단계**에서 **SUPABASE_SERVICE_ROLE_KEY** 를 `.env.local` 에 넣었는지 확인합니다.
- **service_role** 키를 넣었는지 확인합니다. (anon / public 키가 아닙니다.)
- 값을 넣거나 수정한 뒤에는 **4단계**처럼 서버를 **한 번 끄고 다시** 켰는지 확인합니다.

### SQL 실행 시 “relation "users" does not exist” 에러가 나요

- 아직 데이터베이스 테이블을 만든 적이 없다는 뜻입니다.  
  **먼저** `docs/database-schema.sql` 파일 **전체**를 Supabase SQL Editor에 붙여 넣고 **Run** 한 뒤,  
  그다음에 다시 `003-guest-test-user.sql` 을 실행하세요.

---

## 나중에 로그인을 켜고 싶을 때

실제 로그인만 사용하고, “로그인 없이 테스트”를 끄고 싶다면:

1. `.env.local` 을 엽니다.
2. **GUEST_TEST_USER_ID=guest-test** 줄을 **지우거나** 앞에 `#` 을 붙여서 비활성화합니다.  
   예: `# GUEST_TEST_USER_ID=guest-test`
3. 서버를 다시 켭니다.  
   이제 로그인한 사용자만 포인트·기부가 보이고, 비로그인 시에는 예전처럼 “게스트”만 보이게 됩니다.

---

정리하면:  
**1) Supabase SQL로 테스트 사용자 만들기 → 2) Service Role Key 복사 → 3) .env.local 에 두 줄 넣기 → 4) 서버 재시작 → 5) 브라우저에서 5만 P 확인** 순서만 지키면 됩니다.
