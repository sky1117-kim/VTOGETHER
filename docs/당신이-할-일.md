### 3) 환경 변수 (.env) — 단계별로 따라하기

**`.env`가 뭐냐면**  
프로젝트가 “Supabase 주소”랑 “비밀 키”를 알아야 DB에 접속할 수 있습니다. 이걸 코드에 직접 쓰면 위험하니까, **`.env`라는 파일 한 군데에만 적어 두고**, 앱이 그 파일을 읽어서 쓰게 합니다.

---

#### Step A. .env 파일 열기

1. Cursor(또는 VS Code)에서 **프로젝트 맨 바깥 폴더**(`vtogether` 폴더)를 연 상태인지 확인하세요.
2. **`.env`** 파일이 이미 있으면 그걸 **엽니다**.  
   (없다면: `.env.example`을 복사해서 같은 폴더에 붙여넣고, 이름만 `.env`로 바꾼 뒤 열기.)
3. 아래 Step B에서 복사한 값들을 **이미 있는 줄은 확인만 하고, 없는 줄만 추가**하면 됩니다.

---

#### Step B. Supabase에서 값 3개 복사하기

1. 브라우저에서 **https://supabase.com** 접속 후 로그인합니다.
2. 사용 중인 **프로젝트**를 클릭해서 들어갑니다.
3. 왼쪽 맨 아래 **톱니바퀴 아이콘**을 누릅니다. → **Project Settings** (프로젝트 설정) 페이지로 갑니다.
4. 왼쪽 메뉴에서 **API** 를 클릭합니다.
5. 화면에 **Project URL**, **Project API keys** 같은 섹션이 보입니다.

   **첫 번째 값 (Project URL)**  
   - **Project URL** 이라고 써 있는 칸에 `https://어쩌고.supabase.co` 같은 주소가 있습니다.  
   - 그 **전체 주소를 복사**합니다. (옆에 복사 버튼 있으면 그걸로 복사)
   - `.env` 파일에 **한 줄** 추가합니다:
     ```env
     NEXT_PUBLIC_SUPABASE_URL=여기_붙여넣기
     ```
     → `=` 뒤에 복사한 주소를 **따옴표 없이** 그대로 붙여넣습니다.  
     예: `NEXT_PUBLIC_SUPABASE_URL=https://abcdefghijk.supabase.co`

   **두 번째 값 (anon public 키)**  
   - **Project API keys** 안에 **anon** / **public** 이라고 된 키가 있습니다. (긴 문자열)
   - 그 **anon public** 키를 **복사**합니다.
   - `.env`에 **한 줄** 더 추가합니다:
     ```env
     NEXT_PUBLIC_SUPABASE_ANON_KEY=여기_붙여넣기
     ```
     → `=` 뒤에 복사한 키를 붙여넣습니다.

   **세 번째 값 (service_role 키) — 관리자 기능 쓸 때만**  
   - 같은 API 페이지에서 **service_role** 이라고 된 키가 있습니다. (비공개라고 표시됨)
   - **Reveal** 버튼을 눌러 보이게 한 뒤, 그 키를 **복사**합니다.
   - `.env`에 **한 줄** 더 추가합니다:
     ```env
     SUPABASE_SERVICE_ROLE_KEY=여기_붙여넣기
     ```
   - ⚠️ 이 키는 **비밀번호처럼** 취급하세요. GitHub에 올리면 안 됩니다. (`.env`는 보통 올라가지 않게 되어 있음)

6. (선택) 로컬에서 주소가 `http://localhost:3000` 이면 그대로 두고, 아니면 다음도 넣을 수 있습니다:
   ```env
   NEXT_PUBLIC_APP_URL=http://localhost:3000
   ```

---

#### Step C. .env 파일 최종 모양 (예시)

`.env` 파일이 대략 아래처럼 **4~5줄** 정도 되면 됩니다. (실제 값은 각자 프로젝트에 나온 걸로 채우세요.)

```env
NEXT_PUBLIC_SUPABASE_URL=https://abcdefghijk.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpc3MiOi...
SUPABASE_SERVICE_ROLE_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpc3MiOi...
NEXT_PUBLIC_APP_URL=http://localhost:3000
```

- **줄마다** `이름=값` 형태. `=` 앞뒤에 공백 넣지 않기.
- 값에 **따옴표** 붙이지 않기.
- **저장**한 뒤, 서버 다시 실행(`npm run dev`)하면 적용됩니다.

---

**정리**:  
1) `.env` 파일 열기 (이미 있으면 그대로 사용, 없을 때만 `.env.example` 복사해서 `.env`로 만들기)  
2) Supabase → Project Settings → API 에서 URL, anon 키, service_role 키 복사  
3) `.env`에 **없는 변수만** 추가하고 저장 (이미 있는 줄은 그대로 두면 됨)
